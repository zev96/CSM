# 榜单卡片素材规范（共享盘 CLAUDE.md 同步内容）

**用途**：十大排名榜单文的推荐区素材约定。本文件是要同步进共享盘
`D:\家电组共享\DATA\CLAUDE.md` 的内容底稿 —— 共享盘那份是团队 Obsidian 侧的
唯一规范来源，代码只认 vault 本身，两边drift 时以 vault 实际内容为准。

关联：设计方案 `docs/superpowers/specs/2026-07-21-ranked-recommendation-version-groups-design.md`；
可运行示例 `examples/vault/营销资料库/产品模块/空气净化器/竞品推荐内容/`。

## 1. 竞品卡（新增，§3 素材类型加一个值）

`素材类型: 竞品卡`。落在 `产品模块/<产品线>/竞品推荐内容/竞品卡-<型号>.md`。

- **文件名必须含型号**（引擎不强制，靠命名纪律 + 覆盖度检查事后提示）。笔记 id =
  文件名（不含扩展名），全库唯一。撞名的实际后果是两条：索引里同名笔记后者覆盖前者
  （另一张卡直接查不到），以及反馈权重按笔记 id 聚合、两张卡的数据被合成一个桶。
  竞品卡的**重随不会**串到别家竞品（重随先按竞品身份过滤），但主推卡小节与旧格式
  素材的重随是按笔记 id 取的，那里会串。
- frontmatter 必填 `品牌` + `型号`（卡片模式不走文件名兜底）；可选 `层级标签`
  （渲染进标题行，人工填写）。
- 正文 `## 小节名` 分节，节内 `①②③` 放多份候选内容。
- 正文可用 `**加粗**` 突出关键数据，会原样输出。
- 同一竞品可以有多张合格卡，互为候选（随机用一张）。
- 一张「超集卡」写全多个版本要的 H2，即可同时服务多个版本。

## 2. 主推点素材（沿用现有子树，§3 素材类型批量新增）

沿用 `产品模块/<产品线>/<品牌>推荐内容/` 下的既有分层（品牌背书 / 核心技术 /
次要技术…），按下表补 `素材类型` 值。每个值可以有多篇笔记、每篇多个 `①②③`，
两级随机就是「同一个点的多版本内容」。

### 版本1 · 口碑权威型

| 素材类型值 | 建议归属目录 |
| --- | --- |
| 市场口碑数据 | `<品牌>推荐内容/品牌背书/` |
| 品牌赛道定位 | `<品牌>推荐内容/品牌背书/` |
| 全场景适配范围 | `<品牌>推荐内容/品牌背书/` |
| 除醛维度 / 消毒维度 / 过敏原维度 / 体验维度 | `<品牌>推荐内容/核心技术/` |
| 横评总结点评 | `<品牌>推荐内容/品牌背书/` |

「分维度硬核测评」在成文里是一个加粗标题 + 四段正文：模板里配四个小节，只有第一个
填标题名、后三个标题留空（留空 = 续段，只出正文）。

**「留空」只对主推卡成立**：竞品卡的小节名要对应卡片里的 `##` 标题，必须填，留空
保存时会报错。竞品那边要多段就在同一个 `##` 下直接写多段正文。

### 版本2 · 功能拆解型

| 素材类型值 | 建议归属目录 |
| --- | --- |
| 品牌实力 | `<品牌>推荐内容/品牌背书/` |
| 核心技术拆解 | `<品牌>推荐内容/核心技术/` |
| 核心参数 | `<品牌>推荐内容/核心技术/` |
| 智能交互与数显 | `<品牌>推荐内容/次要技术/` |
| 安全与静音 | `<品牌>推荐内容/次要技术/` |
| 颜值与成本 | `<品牌>推荐内容/次要技术/` |
| 适用人群 | `<品牌>推荐内容/品牌背书/` |

语义相近的点（版本1「品牌赛道定位」与版本2「核心定位」）可以复用同一个值省素材，
也可以各建各的隔离风格——引擎不设限，由内容运营决定。

## 3. 治理

`素材类型` 的实际取值早已超出规范列表（现存 21 个，本次再加约 15 个）。两条纪律：

1. 新增值必须同步进共享盘 CLAUDE.md §3 的标准值清单，不要野生新增。
2. 定期清理：素材录入面板的属性列表（读 `/api/vault/attributes`）能看到 `素材类型`
   实际用了哪些值，没有任何模板引用的值应当合并或删除。注意竞品池的「覆盖度检查」
   查的是**卡片里的 H2 小节**，与 `素材类型` 取值治理是两回事。

## 4. 内容公约（写作纪律，引擎不强制）

- 竞品的「分维度硬核测评」「成本与维护」结尾**必带一个明确短板或数据缺失**。
- 竞品「横评总结点评」结尾补一句短板提醒。
- 不给竞品编造参数，不写贬损性表述；主推的参数必须来自产品参数笔记。

## 5. 常见坑

| 现象 | 根因 | 处理 |
| --- | --- | --- |
| 生成时报「榜单需要 N 个竞品，本池可用 M 个」 | 覆盖度不够 / 被同榜单前一个池选走 | 报错里带逐竞品缺料清单，按清单补 H2 |
| 同一款产品在榜单里出现两次 | 型号写成 `X9` 与 `X-9`（连字符不自动合并） | 覆盖度检查会提示「疑似同款」，删掉多余那张卡 |
| 某竞品死活上不了榜 | `品牌`/`型号` frontmatter 漏写，或 H2 名与模板小节名对不上 | 覆盖度检查会显示该卡实有哪些 H2 |
| 加粗没了 | 该素材走的是旧路径（段落/编号列表），不是卡片小节 | 卡片模式才保留加粗 |
| frontmatter 筛选值匹配不上 | YAML 把 `true`/`123` 解析成了布尔/数字 | 值加引号，如 `素材类型: "竞品卡"` |
| 资料库扫描提示「README.md: 缺少 frontmatter」 | 说明性 .md 没有 frontmatter | 无害，这类文件本来就不该进素材索引；不想看到就把说明写进别的格式 |
| 面板说够 N 个、生成却报数量不足 | 同一榜单区前面的竞品池先把名额占走了 | 覆盖度检查不知道跨池排除，按「本区所有池的总需求」核对名册 |
