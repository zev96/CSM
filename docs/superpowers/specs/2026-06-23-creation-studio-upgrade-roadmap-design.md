# 创作台升级 — 总体路线图（设计稿）

- 日期：2026-06-23
- 范围：把现有「模板 + vault 随机组合」的创作台，升级为「可控、接地、按角度组装、品牌型号准确」的 AI 创作台。本稿是**总纲（umbrella）**，拆成 Phase 0–3 + 一个横切质量层；每个 Phase 后续各出独立设计稿（spec → 计划 → 实现）。
- 现状基线（一句话）：`关键词(长尾) + 模板 + skill → 扫 vault → 种子随机采样(随机选笔记+随机选①②变体) → 渲染毛坯文 → LLM 按 skill 润色 → 导出`。标题事后用标题模块公式另生成；逐 block 可 reroll。

---

## 0. 设计原则 & 关键事实

### 0.1 原则（硬约束）

1. **接地优先、可控合规是不可妥协的**：参数/事实只能来自素材库与品牌记忆库，LLM 只负责组织成文，不得新增事实。禁区（绝对化用语 / emoji / 破折号 / 引流话术）必须有机器校验，不能只靠 prompt 文字。
2. **复用现有抽象**：类型化 block、种子可复现采样、reroll、查重/密度、skill 机制全部保留并增强，不推倒重来。
3. **vault 仍是事实中心**：与现有 Obsidian 工作流一致、可版本化、可在 Obsidian 直接编辑。
4. **全部新能力 opt-in、向后兼容**：不传角度 / 无品牌库 = 今天的行为；旧模板、旧请求继续跑（`Template` 已有 `extra="ignore"`、可选字段默认值的先例）。

### 0.2 关键事实（来自真实 vault 实勘 `D:\家电组共享\DATA\营销资料库`，2026-06-23）

- **规模**：230 个 md，单品类（吸尘器），但 `CLAUDE.md` 已规划 4 条产品线（吸尘器 / 宠物空气净化器 / 食物料理机 / 空气净化器），所有模块均按 `<模块>/<品类>/` 分层 → **多品类预留是真的**。
- **vault 已有成熟约定（关键）**：`D:\家电组共享\DATA\CLAUDE.md` 已定义素材类型 10 个标准值、frontmatter 规范、文件命名规范、双向链接规范、①②③变体格式、索引维护规则。→ **本项目的"受控词表"不是从零定义，而是把这份 CLAUDE.md 形式化进 app**；Q3 的"自动建笔记格式"= 遵守这份规范。
- **角度→选材的映射已部分存在**：`关联数据库.md` 已手工建好「人群-痛点-技术-科普 四维矩阵」「情绪钩子映射」「内容流推荐组合」「全局关键词网络」「品牌→型号表（13 品牌 32 型号）」→ **Phase 2 可直接以它为种子**，大幅降险。
- **品牌记忆原料已存在但分散且不规范**：型号事实散落在 `产品模块/吸尘器/产品参数`（33 型号）、`希喂推荐内容`（核心技术/次要技术/品牌背书话术）、`测试项目模块/品牌产品测试结果`（33）；但**产品参数 frontmatter 缺 `品牌`/`型号`（只在文件名里），参数稀疏不一致**（真空度=0、"未说明"），导致 `build_brand_registry` 在真实库上拿不到数据。→ #3 = 归一 + 回填键 + 对账，不是从零造。

## 1. 统一数据模型（贯穿四块的核心）

| 抽象 | 定位 | 来源 / 现状 |
|---|---|---|
| **受控词表 (Taxonomy)** 🆕 | 单一事实来源的枚举：模块、素材类型(10)、人群分类(16)、产品线(4)、文章类型(4)、情绪钩子、语调(口语/专业/极客)、卖点维度。角度选项、自动分类目标、frontmatter 校验全部引用它。 | 形式化 `CLAUDE.md` §3/§10（大部分已存在），新增「语调」「卖点维度」两个面 |
| **品牌型号记忆 (BrandModelMemory)** 🆕 | 每型号一条结构化记忆：规格参数表(接地白名单) / 认证 / 卖点标签(挂证据) / 话术库(维度 × 口语·专业·极客三档) / 痛点话术 / 人群场景句 / 竞争对比 / 总结模板 / 该型号写作禁区。 | 归一 `产品参数` + `希喂推荐内容` + `品牌产品测试结果`；种子从 `家电科普博主.md` 的 CEWEY 块抽 |
| **角度 (Angle)** 🆕 | 可勾选的多面选材意图：①结构（导购/对比/单品/长文 + 标题类型）②卖点（主打防缠绕/长续航/性价比…）③人群（16 类）④语调（口语/专业/极客）。提供「预设组合」避免裸勾选的组合爆炸。 | 结构=`框架模块`；人群=`用户人群`(16)；卖点=`科普/挑选攻略`维度+`希喂核心/次要技术`；语调=新增 |
| **Skill（拆层 + 角色化）** ♻️ | frontmatter 加 `role: persona \| humanize \| platform`；生成后按角色顺序跑多道 pass（人设→去AI味→平台适配）。品牌事实迁出 skill 进记忆库。 | 现有 skill `.md` 机制（`role` 为新增字段） |
| **结构化笔记写入器 (Vault Writer)** 🆕 | App 内把结构化数据渲染成规范 `.md`：自动写对 frontmatter / 命名 / ①②③变体 / 双向链接 / 更新索引文件；diff 预览、不覆盖同名、可回滚。**让 app 成为 CLAUDE.md 规范的执行者**。 | 今天 vault 是只读，无写能力 → 全新（Phase 1 引入，Phase 3 复用） |

**关系**：`模板(结构骨架) + 角度(选材意图) + 品牌型号记忆(事实+话术) + 人设/去AI味 skill 链(文风)` → 生成。泛用内容（引言/科普/标题/人群痛点）走素材笔记；品牌特定内容走记忆库；受控词表是所有分类/选项/校验的单一来源。

### 1.1 前端信息架构（IA）

新增**一级导航入口「素材库」**（`LeftNav.NAV_TOP`，与 工作台/创作区/监测中心/数据中心/引流/模板库/小红书 平级），作为 vault 内容管理的统一工作域。内部分 tab：

- **浏览**：按 模块/品类/素材类型 筛选 vault 笔记 + 查重（今天 vault 在 app 里无任何入口，只能在设置配路径）
- **录入**：Phase 3 的 AI 拆条 → 分类 → 审 → 入库
- **品牌型号**：Phase 1 的记忆库

原则：**录入不单独占一级入口；品牌型号库也收在此入口下**，避免导航从 7 个膨胀成 9、10 个。「设置」仅保留 vault 路径等配置；"重建索引"等日常操作迁入素材库页。图标避免与模板库的 `library` 撞（用 database/folder 类）。

## 2. 路线图（重排序后：先地基，再头牌，最后扩库）

### Phase 0 — 地基对齐（快赢，低风险，先做）

- **受控词表形式化**：把 `CLAUDE.md` §3/§10 的素材类型/人群/产品线/文章类型形式化为 app 内枚举（配置文件），新增「语调」「卖点维度」。作为角度选项、分类目标、校验依据的**单一来源**。
- **数据对账**：脚本从文件名回填 `产品参数` 的 `品牌`/`型号` frontmatter（让 `build_brand_registry` 在真实库非空）；扫出参数空洞/冲突清单交付给用户；修 `brand_registry`。
- **禁区 lint（确定性质量门禁，快赢）**：扫成稿的 绝对化词 / emoji / 破折号 / 双引号 / 引流话术 → 标红 + 一键清 + 可拦导出。
- **验收**：词表枚举驱动至少一个 UI 下拉；`brand_registry` 在真实 vault 返回 33 型号；禁区 lint 能抓出既有违规并清除。

### Phase 1 — 品牌型号记忆库（地基）

- 把分散的 `产品参数` + `希喂推荐内容`(核心/次要技术/品牌背书) + `品牌产品测试结果` **归一**成每型号一条结构化记忆（见 §1）。
- **落地** = vault 笔记（新「品牌记忆模块」或增强现有 `希喂推荐内容`，Phase 1 spec 定），遵守 `CLAUDE.md` 约定。
- **注入机制**：生成锁定某型号时注入其相关切片（命中卖点 + 对应维度话术 + 对应语调档 + 禁区）；prompt 强约束"参数只能用白名单"；**事实核对**：成稿数字 ∉ 白名单 → 标红。
- **skill 角色化**：加 `role` 字段；把 `家电科普博主.md` 拆成「人设」+「去AI味」两个 skill，品牌事实迁入记忆库。
- **Q3 写入器 v1**：App 内「品牌记忆表单」→ 生成规范笔记（frontmatter/命名/双链/索引），diff 预览 / 不覆盖 / 可回滚。
- **信息架构（IA）**：新增「素材库」一级导航入口（见 §1.1），Phase 1 先立入口 + 放「品牌型号」tab，它将成为 vault 内容管理的统一工作域。
- **验收**：指定型号生成的参数 100% 来自记忆库；事实核对能抓出注入外的数字；CEWEY 块从 skill 清出后输出不退化；「素材库」入口可见、品牌型号 tab 可增删改型号记忆。

### Phase 2 — 标题+角度 智能组装（头牌，混合路线）

- **输入** = 标题（可留空由系统按角度+关键词生成）+ 角度（4 面可勾选 + 预设组合）+ 型号/品类。
- **混合**：确定性采样锁住"事实块/参数/话术"接地；LLM 按 标题+角度 选材排序、串接成文、标题领衔；**不得新增事实**。
- **角度→选材映射**以 `关联数据库.md` 的四维矩阵为种子。
- 变体**按语调选**（不再纯随机）；**reroll 跟随角度的有效 filter**（filter 存入 plan / job 缓存）。
- **skill 链执行**：把"单 skill 润色"换成"按 role 顺序多 pass（人设→去AI味→平台适配）"；UI 显示调用次数/成本，支持逐 pass 预览与单独重跑。
- **验收**：同一关键词换角度 → 结构/选材/侧重明显不同且与标题咬合；参数仍接地；reroll 不跑偏角度。

### Phase 3 — 素材入库 + 自动归类（扩库）

- App 内粘贴/导入文章 → AI 拆条（长文→原子素材）→ 按受控词表分类（素材类型/模块/人群/品类）→ 抽 frontmatter（核心关键词/情绪钩子/品牌型号）→ 生成①②变体 → **「待确认入库方案」** → 用户逐条审 → **复用 Phase 1 写入器**写回（自动维护双链 + 索引）。
- 也可导入产品资料 → 结构化进品牌记忆库（与 Phase 1 闭环）。
- 审阅人机工学：批量接受 / 按 AI 置信度排序 / 只看低置信度。
- **信息架构（IA）**：在「素材库」入口（§1.1）补齐「录入」+「浏览/查重」tab；「设置」只保留 vault 路径配置，"重建索引"等日常操作迁入素材库页。
- **验收**：粘贴一篇软文 → 自动正确拆条归类 + 命中型号 → 确认落库 + 索引/双链自动更新 → 重新生成能用上。

### 横切质量层（#4，拆进各 Phase）

禁区 lint（P0）、去AI味/人设 skill 角色化（P1 拆 + P2 链执行）、事实核对（P1）、批量按角度差异化（P2，`/batch` 已有壳）。

## 4. 必须正视的技术点

1. **变体语调标记**：今天 `note.variants` 只是 `list[str]`，"按语调选变体"需知每条变体语调。品牌话术库天然分三档（免费）；泛用素材笔记要补语调标记（影响 `note_parser` 变体元数据）。Phase 2 spec 定。
2. **reroll 跟随角度**：动态 filter 必须存到 plan / job 缓存，否则 reroll 会从原始池抽到不符角度的素材。
3. **写入器要维护双链 + 索引**：`CLAUDE.md` §5/§11 要求每篇笔记有返回上层双链、且要在索引文件登记，否则 Dataview/图谱失联。写入器不能只落文件。
4. **生成每次全量重扫 vault**：[generate_service.py:156](sidecar/csm_sidecar/services/generate_service.py:156) 每 job 都 `scan_vault` 全量重扫，未用 `vault_service` 缓存索引；库做大后会慢，纳入增量/缓存索引。
5. **品牌/型号键缺失 + 参数稀疏**：Phase 0 回填键；事实核对要容忍"未知"（不编造填空）。
6. **数据自相矛盾**：同一型号在不同文件参数打架 → 品牌记忆库作为唯一真相源，对账清理冲突，旧处停止当事实源。

## 5. 成功指标（每 Phase 落可量化验收）

- **事实准确率**：抽样型号各生成 N 篇，参数错误数 → 0。
- **合规通过**：禁区命中 → 0；去AI味评分达标。
- **出稿效率**：单篇人工修改时长 ↓。
- **覆盖**：角度（结构×卖点×人群×语调）× 型号 的组合可达且产出有差异。

## 6. 后续升级（Phase 4+ 远期）

- **反馈学习闭环**：记录保留/改写的变体·型号·角度组合 → 学偏好、自动排序候选。
- **一键平台分发**：成稿 → 小红书/知乎/公众号风格（复用 XHS 编辑器 + 各平台 converter skill）。
- **事实更新传导**：型号参数变更 → 提示更新记忆库 + 标记受影响历史文章可重生成。
- **横评自动化**：`test_framework` 已有基础 → 选 N 型号自动出对比文 + 测试数据对齐。
- **批量出稿 + 自动评分选优**：禁区 lint + 去AI味评分当筛选器。
