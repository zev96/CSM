# 竞品卡（榜单卡片模式素材）

十大排名榜单文的竞品素材形态：**每个竞品一张卡**。同目录下的旧格式
`竞品-<型号>.md`（整篇一段推荐理由）继续被旧模板消费，两者互不干扰。

## 文件命名

```
竞品卡-<型号>.md        ← 例：竞品卡-欧瑞达X9.md
```

**文件名必须带型号**，这是硬约束不是习惯：笔记 id = 文件名（不含扩展名），
全仓按它寻址——重新随机某一段、反馈权重统计都靠它。两个竞品文件夹里各放一个
`口碑.md`，重随就会串到别家竞品的内容，权重统计也会合成一个桶。

## frontmatter

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `品牌` | 是 | 用于识别竞品身份，会做别名折叠（米家→小米） |
| `型号` | 是 | 同上。写法可带可不带 `竞品-` 前缀，引擎会归一化 |
| `素材类型` | 是 | 固定写 `竞品卡`，模板靠它筛选，避免混入旧格式笔记 |
| `层级标签` | 否 | 渲染进标题行的 `{tier}`，如「热门品牌」「性价比之选」。人工填写 |
| `产品` | 否 | 产品线，便于人工检索 |

品牌/型号缺一不可：卡片模式不走文件名兜底（兜底会把 `口碑.md` 这种无意义
文件名当成型号，造出幽灵竞品）。

## 正文

用 `## 小节名` 分节，一个小节 = 榜单卡里的一个「点」。模板里配的小节名与这里的
H2 不必逐字一致（会做宽松匹配），但**不要用会互相包含的名字**。

小节内用 `①②③` 放多份候选内容，生成时随机抽一份——这就是「同一个点的多版本
内容」。同一个点想再多几种写法就继续往下加圈码。

正文里的 `**加粗**` 会原样保留，用来突出 `**550m³/h**` 这类关键数据。

## 一张卡服务多个版本

不同版本的推荐区要的点不一样（版本1 要「市场口碑数据/品牌赛道定位/…」，版本2 要
「核心定位/净化性能/…」）。把两套点的 H2 都写进同一张卡即可——哪个版本要哪几节，
由模板的小节配置决定，卡只要覆盖到就有资格进那个版本的名册。本目录的示例卡就是
这种「超集卡」。

不想混在一张里，也可以每版本各写一张卡（文件名区分，如 `竞品卡-欧瑞达X9-功能版.md`）：
同一竞品的多张合格卡**互为候选**，生成时随机用一张。

## 内容公约

- 「分维度硬核测评」「成本与维护」这类竞品测评节，**结尾必须带一个明确短板或数据
  缺失**（引擎不强制，靠写作纪律）。
- 「横评总结点评」结尾补一句短板提醒。
- 不要写主推品牌的坏话，也不要给竞品编造参数。

## 自检

模板编辑器的竞品池面板上有「覆盖度检查」按钮：能看到哪些竞品够格上榜、谁缺哪一节、
每个小节实际绑到了哪个 H2、文件名有没有撞车、型号写法是不是分裂成了两个竞品。
补完素材记得点一次。
